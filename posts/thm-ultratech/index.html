Le nmap a pris des années mais pour une fois il est intéressant. On a un ftp sur le port 21, une API Express qui tourne sur le port 8081, puis un serveur http sur le port 31331 (pourquoi?)

La page d’accueil du site ressemble à ça:

/robots.txt nous montre qu’il existe un utech_sitemap.txt. En s’y rendant, on découvre la page /partner.html (partie franchement inutile de la room).

On va sur le /partner puis on ouvre devtools, et on voit ceci das le network

Toutes les ? secondes, le site fait une requête vers son port 8081: L’Api Express

Foothold (Command injection on API)⌗

En se rendant sur ultratech.thm:8081/ping, le site nous retourne “name or service not known”

A piori, l’api éxecute localement un ping en fonction du paramètre qu’on lui donne, ce qui expliquerait pourquoi ça n’a pas fonctionné: le site n’a pas ultratech.thm dans son /etc/hosts et donc la commande ne peut pas fonctionner

Pour s’assurer de ça, on peut remplacer le paramètre ip par 127.0.0.1 (une ip que le serveur connaît) pour voir si ça fonctionne:

Et on voit bien que la commande fonctionne. On peut donc partir sur une command injection avec un payload assez simple:

ip=127.0.0.1;`ls`

On voit un seul fichier: utech.db.sqlite. On va utiliser la commande cat pour lire son contenu

On a quelque chose qui ressemble à des identifiants, on va tenter de crack les hash que ça nous a donné:

r00t:f357a0c52799563c7c7b76c1e7543a32

admin:0d0ea5111e3c1def594c1684e3b9be84

On va tester avec https://crackstation.net et le premier hash (md5)

On a le mot de passe de r00t:n100906 qu’on va utiliser pour se ssh

ssh r00t@ultratech.thm
